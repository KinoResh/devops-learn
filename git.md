## git init
Створює новий репозиторій в поточній папці.
Створює приховану папку `.git` де Git зберігає всю історію.

**Використання:**
```bash
git init


## git config
Налаштування Git — імʼя, email, поведінка.

**Використання:**
```bash
# Встановити імʼя
git config --global user.name "Твоє Імʼя"

# Встановити email
git config --global user.email "email@gmail.com"

# Запамʼятати токен
git config --global credential.helper store

# Переглянути всі налаштування
git config --list
Рівні налаштувань:
--global - для всіх проектів на компʼютері
--local - тільки для одного проекту
--system - для всіх користувачів компʼютера
Пріоритет: system < global < local

## git reset
Прибирає файли з індексу назад в робочу папку.
Тобто скасовує git add.

**Використання:**
```bash
# Прибрати конкретний файл з індексу
git reset notes.txt

# Прибрати всі файли з індексу
git reset
Приклад:
Зробив git add . і випадково додав не той файл.
git reset прибирає його з індексу.
Файл залишається з усіма змінами — просто не піде в коміт!
.gitignore
Файл в якому пишеш що Git повинен ігнорувати.
Як створити:
touch .gitignore
nano .gitignore
Що писати всередині:
secrets.txt        # ігнорувати конкретний файл
*.log              # ігнорувати всі файли з розширенням .log
node_modules/      # ігнорувати всю папку
Коли використовувати:
Паролі і токени — ніколи не пушити на GitHub!
